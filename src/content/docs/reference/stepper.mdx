The `<Stepper>` and `<Step>` shortcodes render multi-step procedural content with consistent on-brand visuals. Both are pure server-side components with no client JS.

## Vertical (default)

`<Stepper>` defaults to a vertical layout. Each `<Step>` takes a required `title` prop and a slot for the body content.

```mdx
<Stepper>
  <Step title="Install Qiln">
    Run the installer:
    `qiln install --target bare-metal`
  </Step>
  <Step title="Configure your blueprint">
    Edit `qiln.toml` to point at your GPU pool.
  </Step>
  <Step title="Start the daemon">
    Run `qiln start`. The HTTP API comes up on `:8080`.
  </Step>
</Stepper>
```

<Stepper>
  <Step title="Install Qiln">
    Run the installer:
    `qiln install --target bare-metal`
  </Step>
  <Step title="Configure your blueprint">

  </Step>
  <Step title="Start the daemon">
    Run `qiln start`. The HTTP API comes up on `:8080`.
  </Step>
</Stepper>

## Horizontal

Pass `orientation="horizontal"` for a left-to-right layout. The stepper automatically collapses back to vertical below the `md:` breakpoint (768px), so it never breaks on narrow viewports.

```mdx
<Stepper orientation="horizontal">
  <Step title="Install">Get Qiln onto a host.</Step>
  <Step title="Configure">Point at your GPU pool.</Step>
  <Step title="Start">Run the daemon.</Step>
</Stepper>
```

<Stepper orientation="horizontal">
  <Step title="Install">Get Qiln onto a host.</Step>
  <Step title="Configure">Point at your GPU pool.</Step>
  <Step title="Start">Run the daemon.</Step>
</Stepper>

Horizontal works best with three to four short-title steps. For longer titles or richer bodies, stay with vertical.

## Body content

The `<Step>` body slot accepts any MDX content: paragraphs, code blocks, links, and even nested shortcodes like `<Callout>`. The first and last child margins are trimmed automatically so step bodies sit flush against their titles.

```mdx
<Stepper>
  <Step title="Check prerequisites">
    You'll need a Linux host with NVIDIA drivers installed.

    <Callout type="warning" title="GPU driver version">
      Qiln requires CUDA 12.0 or later.
    </Callout>
  </Step>
  <Step title="Run the installer">
    `qiln install --target bare-metal`
  </Step>
</Stepper>
```

## Single step

A `<Stepper>` with one `<Step>` renders one circle with no trailing connector line. Safe to wrap a conditional single step without special-casing the count.

```mdx
<Stepper>
  <Step title="Restart the service">
    `systemctl restart qiln`
  </Step>
</Stepper>
```

<Stepper>
  <Step title="Restart the service">
    `systemctl restart qiln`
  </Step>
</Stepper>

## Props

### `<Stepper>`

Extends standard `<ol>` HTML attributes.

| Prop          | Type                          | Default      | Description                                                            |
| ------------- | ----------------------------- | ------------ | ---------------------------------------------------------------------- |
| `orientation` | `"vertical" \| "horizontal"`  | `"vertical"` | Layout direction. Horizontal collapses to vertical below 768px.        |

### `<Step>`

Extends standard `<li>` HTML attributes.

| Prop    | Type     | Description                                |
| ------- | -------- | ------------------------------------------ |
| `title` | `string` | Required. The step heading text.           |

## Accessibility

The shortcode renders as a semantic `<ol>` of `<li>` elements, so assistive tech announces "list with N items" and identifies each step by its position. The numbered visual marker is generated by CSS `counter()` and does not affect the underlying list semantics. The horizontal orientation does not change reading order: keyboard and screen-reader navigation still walks the steps in DOM order.
